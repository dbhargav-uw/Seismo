# Seismo — Seismic Viability

A location-aware **conceptual** seismic-viability screening tool for Southern California built on the Scripps reduced-order seismogram dataset. Pick a SoCal site → spec a parametric structure → choose a canned scenario → get an explainable score.

> **Conceptual screening only.** This is not licensed engineering software, not a permit/code-compliance tool, and not a substitute for a structural engineer.

## Repo layout
```
app/
  backend/    FastAPI + Pydantic v2 (port 8000)
  frontend/   Vite + React + TS strict + Tailwind + Mapbox + R3F (port 5173)
data/
  raw/scripps_rom/     Immutable source files (38 MB .npy + demo notebook)
  processed/scripps_rom/   Generated by scripts/, gitignored

  check_data_layout.py
  generate_synthetic_metadata.py    Synthetic SoCal placeholder coords
  preprocess_rom.py                 Per-trace summary metrics + decimated previews
  extract_demo_scenarios.py         3 percentile-picked scenarios
docs/                Architecture, implementation plan, data layout
```

## Prerequisites
- Python 3.11+
- Node 18+ (npm)
- The map ships with a free CartoCDN `dark-matter` vector style — no token required. To swap basemaps, set `VITE_MAP_STYLE` in `app/frontend/.env` to any MapLibre-compatible style URL (Carto's `voyager`/`positron`, MapTiler with a key, or a Mapbox style URL with your token).

## Quickstart

```bash
# 1. Set up Python venv + script deps
python3 -m venv .venv
.venv/bin/pip install -r scripts/requirements.txt

# 2. Build the data spine (validates raw, emits processed + scenarios)
.venv/bin/python scripts/check_data_layout.py
.venv/bin/python scripts/generate_synthetic_metadata.py
.venv/bin/python scripts/preprocess_rom.py
.venv/bin/python scripts/extract_demo_scenarios.py

# 3. Viability backend deps + boot
.venv/bin/pip install fastapi 'uvicorn[standard]' 'pydantic>=2.7' pydantic-settings httpx pytest
PYTHONPATH=app/backend .venv/bin/python -m uvicorn app.main:app --reload --port 8000
#   curl http://127.0.0.1:8000/api/health  →  {"status":"ok"}

# 4. OpenSees backend (separate venv with openseespy + Python 3.10–3.11)
#    Lives in the sibling repo opensees-structure-analysis-starter-fixed/.
#    The Viability backend POSTs to it for every Run viability check.
cd ../opensees-structure-analysis-starter-fixed
python3.11 -m venv .venv && .venv/bin/pip install -e ".[dev]"
.venv/bin/uvicorn app.backend.main:app --port 8001 --reload

# 5. Frontend (in a third terminal)
cd app/frontend
cp .env.example .env             # optional: paste your Mapbox token
npm install
npm run dev                      # http://127.0.0.1:5173
```

**Demo without OpenSees**: set `SEISMO_SIMULATION_BACKEND=placeholder` before launching the Viability backend. The Run flow then uses the deterministic rules-only scoring path; no second backend required.

Then in the browser:
1. Pick a SoCal point (Mapbox click, or fallback lat/lon, or click a receiver row).
2. Tweak the structure form (5 stories / 3 m / 20×20 m default).
3. Pick a scenario (`Light` / `Moderate` / `Strong`).
4. Click **Run viability check** — the result panel shows a 0–100% score with site hazard / ground failure / structural response / uncertainty breakdown plus top drivers.

## Honest disclaimers in the data
Every generated artifact ships with `synthetic_for_demo: true`. The 16 receiver lat/lons are deterministic placeholders centered on the LA basin (seed=42), and the 500 source offsets are a synthetic 10×10×5 (Δℓ,Δw,Δz) grid that stands in for the unavailable `source_locations.csv` referenced by the Scripps demo notebook. The result panel surfaces a "conceptual screening" pill on every run.

## What's next
- **Score philosophy** — the 4-component weighted formula is unchanged from Milestone 1; only the inputs are now real (drift comes from OpenSees instead of being fabricated). Revisit the weights and the constant uncertainty penalty once we have data on which components actually drive useful screening calls.
- **Mode-shape animation** — the per-story drift profile is now in the response; the next batch can lerp the slabs in `StructurePreview3D` to show the deflected shape after a Run.
- **Ground-motion viz** — surface a small waveform/spectrum widget in the Scenario drawer so the user sees what their pick actually looks like.
- **Comparison mode** — two structures, one scenario, side-by-side. Re-uses every existing component.
- **ML surrogate** — with real EDP labels from the OpenSees pipeline, an LightGBM regressor can predict drift in milliseconds; OpenSees verifies in seconds.
